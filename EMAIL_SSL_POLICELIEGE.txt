Objet: Problème de configuration SSL sur policeliege.be - Chaîne de certificats incomplète

Bonjour,

Nous rencontrons un problème lors de la connexion HTTPS à policeliege.be qui empêche l'intégration avec des applications tierces.

PROBLÈME RENCONTRÉ:
Erreur SSL: "certificate verify failed: unable to get local issuer certificate"

CAUSE PROBABLE:
La chaîne de certificats SSL est incomplète. Le serveur n'envoie pas le(s) certificat(s) intermédiaire(s) nécessaires pour valider le certificat du site.

DIAGNOSTIC RAPIDE:
Testez votre configuration sur: https://www.ssllabs.com/ssltest/analyze.html?d=policeliege.be

SOLUTION RECOMMANDÉE:
Configurez votre serveur web pour envoyer la chaîne complète de certificats (certificat du site + certificats intermédiaires).

Pour Apache 2.4.8+:
  SSLCertificateFile /path/to/fullchain.crt

Pour Nginx:
  ssl_certificate /path/to/fullchain.crt;

VÉRIFICATION POST-CORRECTION:
openssl s_client -connect policeliege.be:443 -CApath /etc/ssl/certs/
→ Devrait afficher "Verify return code: 0 (ok)"

IMPACT:
- Empêche l'intégration avec des systèmes automatisés
- Peut afficher des avertissements de sécurité aux utilisateurs
- Affecte les outils de monitoring et d'indexation
